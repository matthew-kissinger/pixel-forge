## Decision

Octa first, VAT fallback.

The first implementation slice should be a WebGL2 animated octahedral impostor array: one character, one clip, 6x6, 7x7, or 8x8 view grid, 8-16 sampled frames, instanced quads, and a `DataArrayTexture` runtime path with a packed 2D atlas fallback. VAT proxy remains the explicit second-best path if angle snapping, overdraw, palette/KTX2 encoding, or WebGL2 texture-array behavior fails.

- Tool reality check: there are tools for this, and the pipeline should use them. Meshy exposes API rigging and animation for textured humanoid GLBs and returns rigged character outputs plus basic animation options ([Meshy rigging API](https://docs.meshy.ai/en/api/rigging-and-animation)). - Gear attachment correction: research-backed rule is rigid gear as bone-attached or single-joint-skinned geometry, while body markings should be projected or split onto the source body triangles. Three.js supports node parenting/attachment through `Object3D` and bones ([Object3D](https://threejs.org/docs/pages/Object3D.html), [SkinnedMesh](https://threejs.org/docs/api/en/objects/SkinnedMesh.html)); glTF skinning is driven by `JOINTS_0` and `WEIGHTS_0` attributes ([glTF 2.0](https://registry.khronos.org/glTF/specs/2.0/glTF-2.0.html)); Three's `DecalGeometry` is a projection option for details but can distort around corners ([DecalGeometry](https://threejs.org/docs/pages/DecalGeometry.html)). The generator therefore removed floating planes and forced the current pass back to clean body zones plus small skinned headgear.

## Locked Constraints

- No Blender in the TIJ runtime or automated build pipeline unless explicitly approved. Blender/Rigify remains acceptable as an open-source authoring/export tool for rigging spikes.
- No new heavy runtime dependency around 5 MB.
- Target per-character full LOD envelope around 30 MB.
- Pixel Forge baker side uses Three.js 0.184 headless with Playwright Chromium.
- TIJ consumer remains Three.js 0.184 and should start with WebGL2, not WebGPU.
- Use existing kiln static-impostor harness, clip resolver, gallery, and pipeline orchestration patterns where possible.
- Keep business logic in `@pixel-forge/core`; adapters remain transport layers.

## Open Risks

- Angle snapping may be visible for moving soldiers at TIJ camera distances.
- Quad overdraw may dominate when thousands of alpha-tested soldiers overlap.
- WebGL2 `DataArrayTexture` behavior may be inconsistent enough to require the packed 2D atlas path immediately.
- R8 palette-index output may be harder than expected without Horde's unpublished baker tooling.
- Optional depth or normal layers may be needed for animated-soldier lighting, pushing storage over the target envelope. Static vegetation is separate: production vegetation normals are required and must be budgeted explicitly.
- Missing or weak shoot clips for NVA/VC remain a content issue independent of the rendering technique.
